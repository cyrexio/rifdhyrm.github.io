üéØ SOC Investigation Best Practices

Golden Rules of Investigation

  • Always check session state first - Understand the context
  • Document everything - Timestamps, actions, findings
  • Preserve evidence - Take screenshots, export logs
  • Verify with users - Contact via phone, not email
  • Think attack chain - Is this part of larger campaign?
  • Check for lateral movement - Did attacker move to other systems?

Evidence Collection Checklist

  • ‚úÖ Network traffic captures (PCAP files)
  • ‚úÖ System logs (Windows Event Logs, Syslog)
  • ‚úÖ Memory dumps (if malware suspected)
  • ‚úÖ File hashes (MD5, SHA256)
  • ‚úÖ Screenshots of alerts and configurations
  • ‚úÖ User interview notes with timestamps
  • ‚úÖ IOCs (IPs, domains, file hashes, emails)

Common False Positives

  • Failed logins - Forgotten passwords, mobile device syncing
  • Recently registered domains - New CDN endpoints, legitimate services
  • Process anomalies - Software updates, admin tools
  • Geographic anomalies - VPN users, traveling employees
üîç Key Windows Event IDs Reference

Security Event IDs (Most Important)

  • 4624 - Successful logon
  • 4625 - Failed logon attempt
  • 4648 - Logon using explicit credentials (RunAs)
  • 4672 - Special privileges assigned to new logon
  • 4720 - User account created
  • 4726 - User account deleted
  • 4728 - Member added to security-enabled global group
  • 4732 - Member added to security-enabled local group
  • 5136 - Directory service object modified (GPO changes)
  • 5140 - Network share accessed
  • 5145 - Network share object checked for access

System Event IDs

  • 7045 - Service installed (potential persistence)
  • 7040 - Service startup type changed
  • 104 - Event log cleared (anti-forensics)
  • 1102 - Audit log cleared

PowerShell Event IDs

  • 4103 - Module logging
  • 4104 - Script block logging (most important)
  • 4105 - Script start
  • 4106 - Script stop
üìû Escalation & Contact Matrix

When to Escalate

  • Immediate Escalation:
    • Active ransomware/wiper malware
    • Domain Admin account compromise
    • Data exfiltration in progress
    • Critical infrastructure systems compromised
  • Standard Escalation (within 1 hour):
    • Multiple related alerts indicating campaign
    • Executive/high-value target compromise
    • Confirmed malware on production systems
    • Suspicious admin activity

CVE-2021-44228
CVSS: 10.0 CRITICAL
Log4Shell - Apache Log4j2 RCE
JNDI features do not protect against attacker controlled LDAP endpoints. An attacker who can control log messages can execute arbitrary code loaded from LDAP servers.
Affected Systems Apache Log4j 2.0-beta9 to 2.14.1
Exploitation Widespread active exploitation
Detection Look for ${jndi:ldap://} patterns in logs
Mitigation Upgrade to Log4j 2.17.1 or later
CVE-2023-23397
CVSS: 9.8 CRITICAL
Microsoft Outlook Elevation of Privilege
An attacker can send a message with extended MAPI properties that trigger automatic authentication against an attacker-controlled SMB share, allowing NTLM hash capture without any user interaction.
Affected Systems Microsoft Outlook (multiple versions)
Exploitation Active exploitation by APT groups
Detection Monitor for SMB connections to unusual IPs
Mitigation Apply Microsoft patches (March 2023)
CVE-2023-34362
CVSS: 9.8 CRITICAL
Progress MOVEit Transfer SQL Injection
SQL injection vulnerability in MOVEit Transfer web application allows unauthenticated attackers to gain unauthorized access to the database and execute SQL statements.
Affected Systems MOVEit Transfer versions before patches
Exploitation Mass exploitation by Cl0p ransomware group
Detection Check for unauthorized database access
Mitigation Apply Progress patches immediately
CVE-2024-3094
CVSS: 10.0 CRITICAL
XZ Utils Supply Chain Backdoor
Malicious code was discovered in XZ Utils versions 5.6.0 and 5.6.1. The backdoor allows remote code execution via SSH authentication bypass. This was a supply chain attack where malicious code was intentionally inserted by a contributor.
Affected Systems Linux systems with XZ Utils 5.6.0/5.6.1
Exploitation Limited - caught early before wide adoption
Detection Check XZ Utils version, monitor SSH logs
Mitigation Downgrade to XZ Utils 5.4.x or earlier
CVE-2024-4577
CVSS: 9.8 CRITICAL
PHP CGI Argument Injection
In PHP versions 8.3.* before 8.3.8, 8.2.* before 8.2.20, and 8.1.* before 8.1.29, when using Apache and PHP-CGI on Windows, a malicious actor can pass command-line arguments to PHP binary, leading to remote code execution.
Affected Systems PHP 8.1.*, 8.2.*, 8.3.* on Windows with Apache
Exploitation Active exploitation in the wild
Detection Monitor for unusual PHP-CGI process arguments
Mitigation Update to PHP 8.3.8, 8.2.20, or 8.1.29

üü† HIGH PRIORITY ALERTS
Recently Registered Domains HIGH
Investigation Steps
  1. Check session state
  2. Check if HTTP communication exists
  3. Check domain reputation using VirusTotal
  4. Review domain registration date
Scenario A: HTTP Communication - Host communicating with malicious domain.
Recommendations:
  • Initiate anti-malware scans
  • Use internal DNS servers
  • Blacklist domain in Firewall
Scenario B: DNS Query Only - No communication established.
Recommendations:
  • Block domain in Firewall preventively
  • No immediate endpoint action required
External Brute-Forced Successful Login HIGH
Investigation Steps
  1. Check session state
  2. Check source IP reputation
  3. Check destination username and hostname
  4. Review failed login attempts before success
Recommendations:
  • Verify login is legitimate
  • Block source IP in Firewall
  • Run anti-malware scans on host
  • Reset user password
RDP Suspicious Logon HIGH
Investigation Steps
  1. Check session state
  2. Check logon IP (source IP) and destination
  3. Review application traffic during RDP session
  4. Check source IP reputation
Recommendations:
  • Verify RDP logon is legitimate
  • Check with source IP and user
  • Block source IP if malicious
Internal Credential Stuffing HIGH
Investigation Steps
  1. Check session state
  2. Check source IP performing testing
  3. Review login activity and traffic patterns
More Info: Anomalously large username/password testing detected.
Recommendations:
  • Check if source IP login activity is legitimate
  • Block internal source IP if not legitimate
  • Review activity after successful logins
Internal SMB Write Anomaly HIGH
Investigation Steps
  1. Check if source and destination are servers
  2. Review monthly traffic patterns
  3. Check volume and frequency
Recommendations:
  • Looks like local server backup
  • Whitelist if legitimate backup operation
  • Block source IP if suspicious
